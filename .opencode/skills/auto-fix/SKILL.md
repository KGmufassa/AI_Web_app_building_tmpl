# Auto Fix

Repair issues detected during testing and validation.

This skill reads validation reports generated by:

- run-tests
- validate-architecture
- drift-check

and attempts to repair problems automatically.

---

# Step 1 — Load Validation Reports

Inspect the following files if present:

validation-reports/test-results.json  
validation-reports/architecture-report.json  
validation-reports/drift-report.json  

If no reports exist, exit with:

"No issues detected."

---

# Step 2 — Classify Issues

Group detected problems into categories.

Test failures

Architecture violations

Specification drift

---

# Step 3 — Apply Repairs

## Test Failures

Example issues:

500 response error  
missing service method  
incorrect return type  

Possible fixes:

- generate missing API endpoint
- create missing service implementation
- fix incorrect response format
- repair broken import paths

---

## Architecture Violations

Example issues:

controller accessing database directly  
incorrect folder placement  
invalid dependency direction  

Possible fixes:

- create repository layer
- move files to correct directories
- update import paths
- enforce layer boundaries

---

## Specification Drift

Example issues:

missing UI page  
missing endpoint  
unused database table  

Possible fixes:

- generate missing components
- generate missing endpoints
- remove unused entities
- update routing configuration

---

# Step 4 — Record Fixes

Write a repair report to:

validation-reports/auto-fix-report.json

Example:

{
  "fixes": [
    {
      "type": "endpoint",
      "action": "generated POST /projects endpoint"
    },
    {
      "type": "architecture",
      "action": "moved projectRepository.ts to repositories folder"
    }
  ]
}

---

# Step 5 — Revalidation

After fixes are applied, the system should rerun validation commands:

run-tests  
validate-architecture  
drift-check  

---

# Output

If fixes were applied:

Issues repaired. Validation will run again.

If no automatic fixes are possible:

Manual intervention required.
